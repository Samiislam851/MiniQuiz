What are the differences between Map,Filter,ForEach and Find?

All of them are array methods . The basic differences can be seen in their functionalities


MAP :

  • Creates a new array by calling a function we write on each element from the original array.
  • Each element is transformed by the given function.
  • The original array is not modified.

Filter :

  • Creates a new array with all elements that pass a test implemented by the provided function.
  • It returns an array with the filtered elements, leaving out any elements that don't pass the test.
  • The original array is not modified.

ForEach :

  • Executes a provided function once for each array element.
  • It does not return a new array, but rather executes a function for each element in the array.
  • The original array is not modified in this method also.

Find :

  • Returns the value of the first element in an array that satisfies a provided testing function.
  • Returns undefined if no matching element is found.
  • The original array is not modified.

What are the differences between Local-Storage, Session-storage, Cookies?

All of them are data storages for storing data in the browser or client side . The basic differences care given bellow :


Local-Storage :

Local storage is a key-value storage that allows data to be stored locally on a user's browser. It stores data permanently until it is deleted by user or our code. Local Storage is useful for storing data that needs to be stored across sessions,tabs.


Session Storage:

Session storage is very similar to Local Storage, a key-value storage mechanism that stores data on the browser. It differs from Local Storage in that, it stores data for the duration of the session. Once we close the browser tab or window, the data stored in Session Storage is gone. Session Storage is useful for storing data that needs to be cleared as soon as the user leaves the website or logs out


Cookies:

Cookies are small text files that are stored on the client-side and are used to store user-specific data, such as login information. Cookies are sent with each HTTP request and can be accessed and modified by both the server-side and client-side. Cookies have an expiration date and can be set to expire after a specific amount of time, or when the user closes the browser.


How does Event loop Work?

When a function is called in JavaScript, it is added to the call stack. The call stack shows the current state of the program. The event loop constantly checks the call stack to see if there are any functions waiting to be executed.If the call stack is empty, the event loop checks the message queue to see if there are any pending tasks waiting to be executed.
When an asynchronous function is called, it is moved outside of the JavaScript engine. After completion it puts a message in the message queue, which is then processed by the event loop.
Then event loop takes the task and moves it from message queue to call stack
Once the function has finished executing, it is removed from the call stack, and the event loop starts over.
